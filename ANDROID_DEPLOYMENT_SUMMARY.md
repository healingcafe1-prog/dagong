## 🎯 배포 방법 선택

### 추천: TWA (Trusted Web Activity) ⭐⭐⭐⭐⭐

**왜 TWA를 추천하나요?**
- ✅ **빠름:** 1-2일 내 완성
- ✅ **저렴함:** 추가 개발 비용 없음
- ✅ **자동 업데이트:** 웹사이트 수정 = 앱 자동 업데이트
- ✅ **PWA 호환:** 이미 PWA로 구축됨
- ✅ **유지보수 간편:** 웹 개발만 하면 됨

**단점:**
- ⚠️ 네이티브 기능 제한 (카메라, 푸시 알림 등은 웹 API 사용)
- ⚠️ HTTPS 도메인 필요

  - ✅ 512x512 PNG 준비 완료
  - 위치: `/static/icons/icon-512x512.png`

- [ ] **Feature Graphic**
  - 1024x500 JPG/PNG
  - Canva로 제작

- [ ] **스크린샷**
  - 최소 2개 (권장 4-8개)
  - 1080 x 1920 (9:16 비율)

- [ ] **법적 문서**
  - 개인정보처리방침: /privacy-policy.html
  - 이용약관: /terms.html

### TWA 설정 🔧

- [ ] **Bubblewrap 설치**
  ```bash
  npm install -g @bubblewrap/cli
  ```

- [ ] **프로젝트 생성**
  ```bash
  bubblewrap init --manifest=https://chadirect.kr/manifest.json
  ```

- [ ] **키스토어 생성**
  ```bash
  keytool -genkey -v -keystore android.keystore ...
  ```

- [ ] **Digital Asset Links 설정**
  - assetlinks.json 생성
  - 웹사이트에 배포
  - 확인: https://chadirect.kr/.well-known/assetlinks.json

- [ ] **AAB 빌드**
  ```bash
  bubblewrap build --release
  ```

  - 앱 이름: 차다이렉트

## ❓ FAQ

### Q1: TWA와 네이티브 앱의 차이는?
**A:** TWA는 웹앱을 앱처럼 포장한 것입니다. 네이티브 앱은 안드로이드 전용으로 개발된 앱입니다. TWA는 빠르고 저렴하지만 네이티브 기능이 제한됩니다.

### Q2: PWA만으로는 안 되나요?
**A:** PWA는 브라우저에서 실행되며 Play Store에 등록할 수 없습니다. TWA를 사용하면 PWA를 Play Store에 등록할 수 있습니다.

### Q3: 앱 업데이트는 어떻게 하나요?
**A:** 웹사이트를 업데이트하면 앱도 자동으로 업데이트됩니다! 별도의 앱 스토어 제출이 필요 없습니다.

### Q4: 네이티브 기능(카메라, 푸시 알림)은 사용할 수 없나요?
**A:** 웹 API를 통해 사용 가능합니다. 예: Web Camera API, Web Push API. 단, 일부 제약이 있을 수 있습니다.

### Q5: 키스토어를 분실하면?
**A:** 앱 업데이트가 불가능합니다. 반드시 안전하게 백업하세요! 분실 시 새 패키지명으로 재등록해야 합니다.

### Q6: 심사 거부되면?
**A:** 거부 사유를 확인하고 수정 후 재제출하면 됩니다. 일반적인 거부 사유:
- Digital Asset Links 미설정
- 개인정보처리방침 누락
- 스크린샷 품질 낮음

### Q7: iOS는 어떻게 하나요?
**A:** iOS는 별도의 프로세스가 필요합니다. PWA로도 충분하지만, App Store 등록을 원하면 Capacitor를 사용해야 합니다.

### Q8: 비용이 더 발생할 수 있나요?
**A:** 기본적으로 $25 이후 추가 비용은 없습니다. 단, 전문 디자인 서비스를 이용하면 추가 비용이 발생할 수 있습니다.
